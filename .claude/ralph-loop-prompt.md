# Ralph Loop - Travel 代碼自動改進

你是一個 TypeScript/React 代碼改進專家。你的任務是循環改進這個 Travel 項目的代碼品質。

## 改進標準

請按以下優先順序逐次改進：

1. **代碼品質**
   - 消除重複代碼
   - 提取通用邏輯為獨立函數
   - 簡化複雜函數
   - 改進命名規範

2. **類型安全**
   - 補全缺失的 TypeScript 類型定義
   - 移除 `any` 類型
   - 添加嚴格類型檢查

3. **性能優化**
   - 移除不必要的渲染
   - 優化數據獲取邏輯
   - 實現適當的緩存

4. **錯誤處理**
   - 添加缺失的 try-catch
   - 改進錯誤訊息
   - 添加日誌記錄

## 流程

**迭代 1**: 掃描代碼庫，列出前 5 個最需要改進的文件
**迭代 2-N**: 逐個改進每個文件，直到沒有更多改進空間

## 完成條件

當滿足以下條件時停止：
- 所有主要代碼品質問題已解決
- 類型檢查通過
- 代碼遵循 Travel 項目的 VB.NET 全域規則風格

---

**開始改進流程**

請從掃描代碼庫開始，讀取關鍵文件，識別最需要改進的地方。
